# 贡献指南

感谢你对《上下文工程实战》教程的关注！我们欢迎各种形式的贡献。

---

## 🤝 如何贡献

### 1. 报告问题

如果你发现了错误或有改进建议：
- 在 [GitHub Issues](https://github.com/yourusername/context-engineering-tutorial/issues) 中创建新issue
- 清楚描述问题或建议
- 如果可能，提供截图或代码示例

### 2. 改进文档

- 修正拼写、语法错误
- 优化表述，让内容更易理解
- 补充示例和图表
- 更新过时的内容

### 3. 贡献代码示例

- 在 `examples/` 目录下添加新示例
- 确保代码可运行
- 添加详细注释
- 提供requirements.txt或依赖说明

### 4. 添加资源

- 在大纲中补充优质资源
- 包含资源链接、类型、核心价值说明

---

## 📝 提交流程

1. **Fork 本仓库**
2. **创建特性分支** (`git checkout -b feature/AmazingFeature`)
3. **提交更改** (`git commit -m 'Add some AmazingFeature'`)
4. **推送到分支** (`git push origin feature/AmazingFeature`)
5. **开启 Pull Request**

---

## ✅ 提交规范

### Commit Message 格式

```
<type>: <subject>

<body>
```

**Type类型**：
- `docs`: 文档更新
- `feat`: 新增功能/章节
- `fix`: 修复错误
- `style`: 格式调整
- `refactor`: 重构
- `example`: 代码示例
- `chore`: 其他修改

**示例**：
```
docs: 完善第5章查询增强内容

添加了HyDE方法的详细说明和代码示例
```

## 📋 文档规范

### 章节结构

```markdown
# 第X章：章节标题

> **学习目标**：列出3-5个具体目标

---

## X.1 小节标题

（内容）

## X.2 小节标题

（内容）

---

**参考资源**：
- 资源1
- 资源2
```

### 代码示例规范

```python
# 文件：example_name.py
# 描述：简要说明

# 导入
import xxx

# 主要代码
def main():
    """
    功能说明
    """
    pass

if __name__ == "__main__":
    main()
```

## 🎨 内容风格

- **通俗易懂**：避免过度学术化
- **循序渐进**：由浅入深
- **实战导向**：理论结合实践
- **中文为主**：专业术语保留英文+中文对照
